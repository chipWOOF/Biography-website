# Battle-of-the-machines
A project that creates a generative adverserial network and uses game theory to gain further insights on security.  OWNER MINH BAO BUI 


---------------------- context ------------------------
Motivation: In a world where AI is being used in almost every single aspect, this experiment explores whether implementing AI into security systems is a safe, beneficial and profitable innovation as compared to traditional static security measures. 


AIM: The aim of this project is to determine if dynamic AI implemented security is more secure than traditional static security measures against a capable and well equipped attacking force, and whether the opportunity cost outweighs the added benefit.

Hypothesis: I hypothesise that the effectiveness of static security measures will be much lower than dynamic security measures due to the use of AI in black hat hacking, and that the addded benefit of implementing an effective dynamic security AI system outweights the cost of its development and maintenance. 

method: To complete a conclusion this project will follow these steps:
1. Create a simulation framework to explore adversarial security strategies.
2. Implement adaptive strategy agents using machine learning or otherwise.
3. Assign BLUFOR and OPFOR to strategy agents, BLUFOR representing defenders, OPFOR representing attackers.
4. Train BLUFOR agents on real world security conventions and strategies.
5. Train OPFOR agents on real world security attacking strategies.
6. Simulate thousands of attack-defense interactions between both agents, repeatedly having both agents fight again and again
7. Analyse trade-offs between detection, cost and attack success rates.
8. Record highest success rate strategies and their conditions.
9. Repeat steps 1-8 with only static security measures
10. Record where static security measures differ in success and failures compared to dynamic AI agent.
11. Analyse both data sets, combining both to create the strongest and most mapped security strategy table


12. This is where it gets interesting, now that we have all these data sets, combine both OPFOR and BLUFOR agents with the static security data set, their strategies, their behaviours, their insights, biases, and knowledge into one agent.
13. using this new agent, Have it play as both the defender, and the attacker.
14. note the results in this new ULTRON agent and the trade-offs between detection, cost and attack success rates.

15. Analyse the difference between ULTRON agent compared to BLUFOR agent and static security. 
16. Analyse the difference between BLUFOR vs static security.
17. Map the results according to effectiveness, cost, time, and ethics.
